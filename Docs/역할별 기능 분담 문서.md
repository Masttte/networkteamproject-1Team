## 협업 원칙

### UI는 한 명이 독점하지 않는다
- **고병희**는 **UI 통합·최종 디자인·게임 플로우 연결**에 집중
- 각 담당자는 **자기 시스템의 디버그/임시 UI를 직접 제작** 가능
  - 예: 플레이어 HP 수치 표시, 미션 진행도 바, 몬스터 상태 표시 등
  - Week 1~2는 임시 UI로 각자 테스트, Week 3 폴리싱 단계에서 고병희가 통합 디자인
- 정식 UI는 고병희가 일관된 톤·레이아웃으로 정리
- **로비 UI, 승패 결과 화면, 타이머** 등 게임 플로우 핵심 UI는 고병희 전담

### 시스템 인터페이스 계약 우선
- 시스템 간 연결은 **인터페이스로 먼저 합의**한 뒤 병렬 개발
- 예: `IInteractor` / `IInteractable`은 Day 1~2에 서명만 확정 → 양쪽이 독립 개발
- 인터페이스 변경 필요 시 팀에 공유

## 네트워크 & 세션 & 연출 (박언약)

### 담당 영역
네트워크 기반 시스템 전반 + 일부 연출. 모든 팀원이 이 위에서 NetworkBehaviour를 작성하므로 **프로젝트의 기초 레이어**.

### 주요 작업
| 작업 | 순위 | 비고 |
|---|:---:|---|
| Unity Netcode + Relay 초기 셋업 | 1 | 프로젝트 시작 전 사전 작업 완료 |
| 세션 생성/접속 (1-1) | 1 | |
| 접속 플레이어 동기화 | 1 | 로비 씬에 연결 |
| 게임 Start 로직 (1-2) | 1 | 4인 정원 체크 |
| A/B 랜덤 할당 + 역할 은닉 | 1 | **핵심 기믹** |
| 로비 → 인게임 씬 전환 | 1 | |
| NetworkBehaviour 공통 베이스 / 권한 유틸 | 1 | 전원이 사용 |
| **A/B 시야 분리 연출 (8)** | **1** | 캐릭터 오브젝트 레이어 컬링(카메라 Culling Mask) 기반, 필요 시 렌더링 담당(이성규) PPS 지원 |

### 다른 영역에게 제공
- **역할 조회 API**: `IsLocalPlayerA()`, `IsLocalPlayerB()` — 전원이 사용
- **NetworkBehaviour 공통 베이스** — 전원이 상속
- **데미지 ServerRpc 패턴** — 플레이어 담당과 협의 후 표준화

### ⚠ 주의
- **권한 모델을 빠르게 확정**해야 나머지 작업이 시작 가능
- **A/B 역할 은닉**은 Owner-only ClientRpc로 구현 (마피아가 누군지 A그룹에게 숨김)
- **호스트 끊김 시 게임 종료** 정책으로 단순화 (호스트 마이그레이션 구현 X)

### 권한 모델
- **AI 로직은 호스트(Server) 단독 실행**

### ⚠ 주의

- **감지 범위 밖 이탈 시 동작** 명확히 — 스폰 위치 복귀? 현 위치 대기? 회의 확정
- **데미지 전달은 플레이어 `IDamageable` 사용**

| 정식 HUD 통합 디자인 | 2 | 각 담당자 임시 UI 통합 후 일관된 톤으로 재작업 |

- **게임 플로우 스크립트가 이 역할의 중심** — UI는 흐름의 표현일 뿐
- **디버그 UI는 각 담당자가 직접 제작** — 고병희는 통합 디자인·최종 폴리싱에 집중
- **HUD는 2단계 접근**: Week 1~2엔 각 담당자의 임시 UI로 테스트, Week 2 후반부터 통합 재작업
- **A용 / B용 HUD 분기** 필수 (마피아는 자기가 마피아인 걸 알아야 함)
- **승패 판정은 반드시 서버에서** — 클라에서 하면 조기 종료 가능
- **호러 장르 사운드 기여도 큼** — 최소 몬스터 근접 사운드는 Week 3 필수

## 역할 간 의존 관계

```
맵 뼈대 ──────────┐
                  ├─→ 전원의 작업 환경
Netcode 셋업 ────┘
                  ↓
      ┌───────────┼───────────┐
   플레이어     몬스터     상호작용
      │           │           │
      └───────────┼───────────┘
                  ↓
         게임 플로우 (흐름 관장)
                  ↓
              연출 · 사운드
```

**블로커 체인**
1. **Day 1-2**: 맵 뼈대 + Netcode 셋업 — 네트워크 담당이 프로젝트 시작 전 상당 부분을 사전 진행하여 **블로커 리스크 완화됨**
2. **Day 3-5**: 각 영역 1순위 프로토타입
3. **Week 2**: 통합 + 2순위 기능
4. **Week 3**: 폴리싱
