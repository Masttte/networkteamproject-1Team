# 에셋 & 퍼포먼스 최적화 가이드

> **프로젝트명:** 비대칭 생존 호러 네트워크 게임 (제목 미정)  
> **버전:** 1.0 | 2026.04  
> **빌드 타겟:** Windows Standalone (itch.io 배포)  
> **작성자:** 이성규 (렌더링 / 에셋 최적화 담당)

---

## 1. 이 문서의 목적

에셋 Import 및 설정 단계에서 참고할 최적화 기준을 정리한 문서입니다. 강제 사항이 아니라 기준점이며, 에셋 성격이나 상황에 따라 유연하게 적용합니다. 판단이 애매하면 렌더링 담당에게 편하게 물어봐주세요.

네트워크 멀티플레이 + 호러 장르 특성을 반영했습니다. 4인 동시 플레이 + 조명/포스트프로세싱 의존도가 높은 장르라, 일반 3D 프로젝트보다 조금 더 세심한 관리가 필요합니다.

---

## 2. 퍼포먼스 참고치

아래는 절대 기준이 아니라 참고 수치입니다. 에셋 배치나 연출에 따라 초과할 수 있고, 실제로 프레임이 안 떨어지면 문제없습니다. 네트워크 게임이라 **호스트 클라이언트에 부하가 더 크게 걸리는 점**은 염두에 둬주세요.

| 항목 | 참고치 | 측정 방법 |
|------|--------|-----------|
| 타겟 FPS | 60fps 이상 | Profiler → CPU/GPU Time |
| SetPass Calls | 200 내외 | Frame Debugger |
| Draw Calls (Batched) | 500 내외 | Game 뷰 Stats 창 |
| 화면 삼각형 수 | 500K 내외 | Game 뷰 Stats 창 |
| 런타임 메모리 | 1GB 이하 | Profiler → Memory |
| 빌드 크기 | 1GB 이하 | itch.io 업로드 제한 |

삼각형 수나 드로우콜은 씬 구성에 따라 달라지니까, 수치 자체보다 "체감 프레임이 떨어지는가"를 기준으로 판단합니다.

> ⚠️ **네트워크 프로젝트 주의**: Profiler로 잡히지 않는 네트워크 지연이 있을 수 있습니다. 로컬에서 60fps여도 Relay 통해 붙으면 체감 프레임이 다르게 느껴질 수 있어요. 멀티 테스트 환경에서 한 번씩 체크해주세요.

## 3. 텍스처

### 3.1 해상도 참고 기준

| 용도 | 권장 최대 해상도 | 비고 |
|------|------------------|------|
| 캐릭터 (플레이어) | 2048 | Albedo, Normal, Mask |
| 몬스터 | 1024~2048 | 근접해서 볼 기회 많음 |
| 미션 오브젝트 (상호작용) | 1024 | 플레이어가 가까이 붙음 |
| 배경 프롭 (대형) | 1024 | 타일링 텍스처 권장 |
| 배경 프롭 (소형) | 512 | |
| UI 요소 | 개별 지정 | |
| 이펙트/파티클 | 256~512 | |

화면에서 작게 보이는 오브젝트에 큰 텍스처를 쓰면 메모리 낭비가 되니 실제 화면 점유 크기를 기준으로 판단합니다.

### 3.2 Import 설정 권장

| 항목 | 권장 설정 | 왜? |
|------|-----------|-----|
| Generate Mip Maps | 3D 오브젝트: On / UI: Off | UI에 Mip Map이 있으면 메모리만 늘어남 |
| Read/Write Enabled | Off | On이면 메모리 2배 사용 |
| sRGB | Albedo/UI: On / Normal/Mask: Off | 컬러 스페이스 맞추기 |
| Alpha Source | 알파 불필요 시 None | 불필요한 알파 채널은 메모리 낭비 |

### 3.3 Sprite Atlas (UI)

UI 스프라이트를 Atlas로 묶으면 Draw Call을 줄일 수 있습니다. 필수는 아니지만, UI 요소가 많아져서 Draw Call이 눈에 띄게 늘어나면 고려해봐주세요. Atlas 구성이 필요하면 렌더링 담당이 서포트합니다.

## 4. 메시

### 4.1 폴리곤 참고 기준

| 용도 | Tris 참고치 | 비고 |
|------|-------------|------|
| 캐릭터 (플레이어) | 5K~15K | |
| 몬스터 | 5K~12K | 근접 시 디테일 중요 |
| 미션 오브젝트 | 500~2K | 상호작용 시 가까이서 보임 |
| 배경 프롭 (대형) | 1K~5K | |
| 배경 프롭 (소형) | 100~1K | |
| 이펙트 메시 | 100~500 | |

에셋 스토어에서 가져온 메시는 이 기준과 다를 수 있는데, 실제 퍼포먼스에 문제가 없으면 그대로 사용해도 괜찮습니다.

### 4.2 Import 설정 권장

| 항목 | 권장 설정 | 왜? |
|------|-----------|-----|
| Read/Write Enabled | Off | On이면 메모리 2배 |
| Optimize Mesh | Everything | 렌더링 효율 향상 |
| Import Cameras | Off | DCC 도구의 카메라가 딸려옴 |
| Import Lights | Off | DCC 도구의 라이트가 딸려옴 |
| Import Visibility | Off | 거의 쓸 일 없음 |
| Import BlendShapes | 필요 없으면 Off | 메모리 절약 |

### 4.3 배칭 참고

- Static 오브젝트는 Inspector에서 Static 체크하면 Static Batching이 적용됩니다.
- SRP Batcher는 동일 셰이더를 쓰면 머티리얼이 달라도 자동으로 배칭됩니다.
- 같은 메시가 많이 배치되는 경우 GPU Instancing을 켜면 효과적입니다.
- 네트워크로 스폰되는 오브젝트(몬스터, 플레이어)는 Static 처리하면 안 됩니다.

## 5. 머티리얼 & 셰이더

### 5.1 머티리얼

- 외형이 같은 오브젝트는 머티리얼을 공유하면 배칭 효율이 올라갑니다.
- 런타임에서 머티리얼 프로퍼티를 바꿔야 하면 `material` 프로퍼티로 인스턴스를 만들어 사용합니다. `sharedMaterial`을 직접 수정하면 같은 머티리얼을 쓰는 모든 오브젝트가 바뀝니다.
- 빈 머티리얼 슬롯이 있으면 핑크색으로 렌더링되니 확인해주세요.

### 5.2 셰이더

- 핑크 머티리얼(셰이더 에러)이 보이면 알려주세요. 빌드에서 문제가 됩니다.
- SRP Batcher 호환 여부는 렌더링 담당이 Frame Debugger로 확인합니다.
- URP 셰이더(Universal Render Pipeline) 사용이 원칙, Built-in 셰이더는 URP에서 핑크로 깨집니다.

### 5.3 호러 장르 조명 & 포스트프로세싱

호러 게임은 조명과 PP가 분위기를 만드는 핵심입니다. 다른 요소를 절약해서라도 PP 비용은 확보합니다.

**조명 설정 권장**

| 항목 | 권장 | 비고 |
|------|------|------|
| Lighting Mode | Mixed (Subtractive / Shadowmask) | 정적 조명 중심 |
| Light Probe | 이동 경로에 배치 | 맵 완성 후 |
| Reflection Probe | 반사 필요 구간만 | 과용 시 빌드 용량 증가 |
| Shadow Distance | 30~50m | 실내 위주라 멀리 안 봐도 됨 |

**포스트프로세싱 비용 참고**

| 효과 | 비용 | 호러 장르 활용 |
|------|------|---------------|
| Bloom | 낮음 | 살짝 걸어서 빛 번짐 강조 |
| Color Grading | 낮음 | 채도 다운, 콜드 톤 기본 |
| Tonemapping | 매우 낮음 | ACES 기본 |
| Vignette | 매우 낮음 | **필수 — 시야 제한으로 긴장감 강화** |
| SSAO | 높음 | 실내 구석 어둠 강조에 효과적이지만 비용 주의 |
| Depth of Field | 중~높음 | 클로즈업/점프스케어 연출 |
| Motion Blur | 중간 | 멀미 유발 가능, 옵션화 권장 |
| Film Grain | 낮음 | 필름 노이즈로 호러 톤 강화 |

**A/B 시점 분리 연출**: 네트워크 담당(박언약)과 협의해서 A그룹/B그룹 시점별로 PP Volume을 다르게 적용할 수 있습니다. 필요시 렌더링 담당이 PP 프로파일 분리 지원합니다.

포스트프로세싱 설정은 렌더링 담당이 관리합니다. 추가하고 싶은 효과가 있으면 같이 논의해요.

## 6. UI / Canvas 최적화

UI 쪽은 인스펙터 설정 몇 개로 꽤 많은 최적화가 됩니다.

### 6.1 Raycast Target

**이것만 해도 크게 효과가 있습니다.**

Image, Text, RawImage 등 UI 컴포넌트는 기본값이 Raycast Target = On입니다. 클릭/터치 입력을 받지 않는 요소(배경 이미지, 장식 텍스트, 아이콘 등)는 Off로 꺼주세요. EventSystem이 매 프레임 모든 Raycast Target을 순회하기 때문에, 필요 없는 요소가 켜져 있으면 그만큼 비용이 늘어납니다.

- **On으로 둘 것**: 버튼, 슬라이더, 입력 필드, 토글 등 실제 인터랙션 요소
- **Off로 둘 것**: 나머지 전부 (HUD 수치, 아이콘, 프레임 등)

### 6.2 Canvas 분리

하나의 Canvas 안에서 요소 하나만 바뀌어도 Canvas 전체가 리빌드됩니다. 자주 바뀌는 UI(HP바, 점수, 타이머)와 거의 안 바뀌는 UI(프레임, 배경)를 별도 Canvas로 나누면 리빌드 범위가 줄어듭니다.

네트워크 HUD(HP, 미션 진행도, 타이머)는 자주 갱신되므로 정적 UI와 분리 권장.

다만 초반부터 무조건 나눌 필요는 없고, 프로파일러에서 Canvas.BuildBatch가 눈에 띄게 잡히면 그때 분리해도 됩니다.

### 6.3 Layout 컴포넌트

HorizontalLayoutGroup, VerticalLayoutGroup 등은 자식이 바뀔 때마다 레이아웃을 재계산합니다. 편리하지만 동적으로 자주 바뀌는 리스트에서 쓰면 부하가 생길 수 있습니다. 정적인 배치에는 RectTransform으로 직접 잡는 게 가볍습니다.

다만 에디터상에서 배치할 때만 사용하고 비활성화하는 편법으로 편리함을 챙길 수도 있습니다.

### 6.4 기타 참고

| 항목 | 권장 | 이유 |
|------|------|------|
| TextMeshPro 사용 | 권장 | Legacy Text보다 렌더링 효율 좋음 |
| 매 프레임 텍스트 갱신 | 피하기 | 값이 바뀔 때만 SetText 호출 |
| 안 보이는 Canvas | 비활성화 권장 | alpha=0이어도 리빌드는 발생 |
| Pixel Perfect | 필요할 때만 On | 매 프레임 레이아웃 재계산 유발 |

## 7. 오디오

호러 장르는 사운드 기여도가 매우 큰 편이라 퀄리티를 우선시하되, BGM과 원거리 SFX는 스트리밍으로 메모리를 확보합니다.

| 항목 | 권장 | 비고 |
|------|------|------|
| BGM 포맷 | .ogg (Vorbis) | 용량 대비 품질 좋음 |
| SFX 포맷 | .wav (짧은) / .ogg (긴) | |
| BGM Load Type | Streaming | 메모리에 전부 올리지 않음 |
| SFX Load Type | Decompress On Load (짧은) | |
| Force To Mono | 용량 절약이 필요한 덜 중요한 SFX에 권장 | 남발하면 사운드 경험 하락, 선택적 사용 |
| Preload Audio Data | 자주 쓰는 SFX: On | |
| 3D 사운드 (몬스터 발소리 등) | Stereo 유지 | 방향감이 긴장감의 핵심 |

## 8. 애니메이션

네트워크 프로젝트 특화 주의사항이 있습니다.

| 항목 | 권장 설정 |
|------|-----------|
| Anim. Compression | Keyframe Reduction (기본) |
| 불필요한 스케일 커브 | 스케일 변화 없으면 Import 설정에서 체크 해제 |
| 루프 설정 | Idle, Run 등 반복 애니메이션은 Loop Time 확인 |
| Animator Controller | 사용하지 않는 Layer/State 정리 |

### 네트워크 애니메이션

- **NetworkAnimator 파라미터 최소화**: 파라미터마다 네트워크 트래픽 발생
- **원격 플레이어/몬스터 애니메이션**: 로컬과 다르게 살짝 지연될 수 있음 (보간 처리는 네트워크 담당과 협의)
- **루트 모션**은 네트워크 동기화가 까다로움. 가급적 in-place 애니메이션에 Transform으로 이동 제어 권장

## 11. 에셋 설정 확인 체크리스트

에셋을 Import하거나 설정을 만질 때 참고하는 목록입니다.

### 텍스처
- [ ] 용도에 맞는 해상도인가
- [ ] Read/Write가 꺼져 있는가
- [ ] UI 텍스처의 Mip Map이 꺼져 있는가

### 메시
- [ ] Read/Write가 꺼져 있는가
- [ ] 불필요한 Import 옵션이 꺼져 있는가 (Camera, Light, Visibility)
- [ ] Static 오브젝트에 Static 플래그가 체크되었는가 (단, 네트워크 스폰 오브젝트는 제외)

### UI
- [ ] 인터랙션 없는 UI 요소의 Raycast Target이 Off인가
- [ ] TextMeshPro를 사용하고 있는가
- [ ] 안 보이는 Canvas가 비활성화되어 있는가

### 오디오
- [ ] BGM이 Streaming 로드인가 (권장 수준, 필수 아님)

### 네트워크
- [ ] NetworkAnimator 파라미터가 실제로 쓰이는 것만 남아 있는가
- [ ] 동기화 불필요한 시각적 요소(로컬 이펙트 등)가 NetworkObject에 포함되어 있지 않은가
