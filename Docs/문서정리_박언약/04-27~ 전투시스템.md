## 전체 구조

```
WeaponSO          (ScriptableObject)        ← 무기 스탯 데이터
IDamageable       (interface)               ← TakeDamage 진입점 (BattleManager.cs 내 정의)

[namespace Battle]
BattleManager     (NetworkBehaviour)        ← 게임 규칙 처리 (Despawn)
Weapon            (NetworkBehaviour)        ← 입력 수신, 공격 판정, RPC 흐름
EntityBase        (NetworkBehaviour, abstract) ← HP, 사망 공통 로직
└─ PlayerEntity   (EntityBase)             ← 플레이어 전용 사망 처리
```
## 각 클래스 역할 요약

### WeaponSO
무기 스탯 보관 ScriptableObject (`damage`, `range`, `cooltime`, `attackClip`).

### Weapon
- `IsOwner` 클라이언트만 입력 수신(`BattleInputReader`) 및 Raycast 실행
- 쿨타임 체크 후 `AttackServerRpc(targetId, damage)` 전송
- 연출은 `AttackClientRpc`에서 처리 [TODO]

### EntityBase
- `CurHp`: `NetworkVariable<int>` — `WritePermission.Server`, 전체 동기화
- `TakeDamage()`: **서버에서만 호출** (`Weapon.AttackServerRpc` 내부)
- `IsDead`: 로컬 bool (서버의 `OnHpChanged`에서 set)
- `onDeath`: **서버에서만 발행**되는 사망 이벤트

### PlayerEntity
- `onDeath` 구독 → `AlertDeath()` → `IsServer` 체크 후 RPC 및 BattleManager 호출
- `NotifyDeathClientRpc()` → `IsOwner` 체크 후 `YouDied()` 실행

### BattleManager
- `OnPlayerDead(PlayerEntity)`: **서버 전용** — `NetworkObject.Despawn()` 처리
- TODO: 팀 사망 처리, 승패 판정 등 추가 가능

## 네트워크 공격 흐름

```
[오너 클라이언트]
  TryAttack() → 쿨타임 검사 → Attack()
    → Raycast로 대상 NetworkObject 탐지
    → AttackServerRpc(targetId, damage) 전송

[서버]
  AttackServerRpc()
    → SpawnedObjects에서 대상 조회
    → IDamageable.TakeDamage(damage)       ← 서버 권위 HP 차감
    → AttackClientRpc 전파

[모든 클라이언트]
  AttackClientRpc()
    → 공격 연출 재생 (사운드, 이펙트 등) [TODO]
```

## 네트워크 사망 흐름

```
[서버]
  TakeDamage() → CurHp.Value = 0
    → CurHp.OnValueChanged (서버에서 동기적으로 실행)
    → OnHpChanged() → IsDead = true → onDeath.Invoke()
    → AlertDeath() [IsServer 체크]
      → NotifyDeathClientRpc() 전송   ← Despawn 이전에 전송
      → BattleManager.DestroyEntity()
          → NetworkObject.Despawn()   ← 모든 클라이언트에서 오브젝트 제거

[클라이언트]
  NotifyDeathClientRpc()
    → IsOwner 체크 → YouDied()        ← 오너 클라이언트만 사망 연출 실행
  (이후) Despawn 처리
```
